¿Qué es position?

Position es un modulo de css, un tipo de display que nos permite poner cualquier elemento del layout donde se nos de la gana, claro esto implica que si no lo hacemos bien terminaremos con elementos en lugares donde no queremos que este.

Veremos unos conceptos para comprender ya todo lo que implica esta propiedad ya que como nos permite manipular el elemento con toda libertad tiene sus reglas.


Flujo:Se refiere a el orden que siguen los elementos como lo vamos escribiendo en el código. Explicando, si ponemos un h1 primero este ira en esa posición y si después ponemos un párrafo este ira después del h1 y asi sucesivamente.

Propiedad Offset

Position:

Es una propiedad de CSS que puede ser static, relative, absolute, fixed o sticky (este ultimo es el mas nuevo) y según el tipo de valor que le apliquemos a esta propiedad el elemento se va acomodar de manera diferente. Generalmente cuando hablamos de display entendemos como funciona por lo simple que es, le indicamos que se muestre como bloque o como linea o ambos o que no se muestre, pero con position la cosa cambia ya que cada valor es un mundo.

Comenzamos sabiendo que todos los elementos por default son position: static, ¿qué significa static? este indica que los elementos sigan su flujo normal como viene en el código; y dicho lo anterior es tanto así que solo se considera elementos posicionados a los demás valores con excepción de static. Ahora para comprender primero estos valores tenemos que ver primero offset .

Offset:

las propiedad offset son los valores que mueven un elemento posicionado. Sus valores son: Top, right, bottom, left, y esos los podemos aplicar solo a los que ya mencionamos que solo se consideran posicionados (relative, absolute, fixed o sticky/ static NO).
Ejemplo:


.title { /*position: relative*/ top: 100px; //Esto no cambio nada ya que esta en static.

Lo importante a entender es que top, bottom, left y right son el lado que se toma de referencia para mover un elemento, y otra cosa importante es que todos los elementos van a tener una referencia desde la cual se van a mover, por ejemplo no es lo mismo un top 100px relative a uno de 100px absolute, ¿Cuál es esa referencia? esta ya depende de el position que usemos.

Position relative

Definición:

Sus características son:


Su referencia en este caso es su posición inicial.

El valor de relative por si solo no altera el layout (geometría) del elemento al que se le aplica.

Al usarlo con las propiedades offset su espacio se reserva.

Las ventajas de relative son muchas ya que podemos alterar el lugar del elemento de una forma limpia a diferencia del primer ejemplo de margin el cual tuvimos que calcular poco a poco como se veía para que se viera bien, ya que con margin se altera todo el layout y con relative no.

Position absolute

Definición

Pierde su posición el el flujo y no reserva su espacio

Si tenia dimensiones fijas estas se mantienen (que el width y tal estén especificados), pero si este tiene dimensiones automáticas se ajustara a su contenido, por ejemplo los elementos bloques tienen un width del 100% por automático pero con absolute esto se cambia a un width que se ajusta a su mismo contenido si es que no se a declarado un width.

Al igual indicando el ultimo punto del texto es que si el elemento es inline este pasa a ser un elemento sin display definido, una especie de inline-block seria lo mas similar ya que se ajusta sus dimensiones, al fin de acabo se altera su flujo.

Su referencia es su ancestro posicionado mas cercano. Con ancestro significa al papá o abuelo o tatarabuelo etc así sucesivamente hasta encontrar un contenedor que tenga una position. El anterior punto es el mas importante ya que nos da el punto del cual parte nuestro absolute, y algo a recalcar es que su referencia no forzosamente tiene que ser un relative también puede ser un absolute o cualquier otro position.
