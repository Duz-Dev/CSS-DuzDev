¿Qué son?

Son selectores que reaccionan en tiempo real detectando la interacción de usuario con algunos elementos. Este selector se mantienen atento para ver si interactuamos con ella, y si es así aplicara los estilos que le hemos declarado.

Funcionan con todos los selectores de CSS, podemos aplicarla en todo básicamente.

No están atados a ningún elemento. Podemos ponerla sola en el código tal cual.

Ejemplo

selector:pseudoclase { propiedad:valor; } //Y en caso de no tener ningun selector :pseudoclase { propiedad:valor; }

Dynamic

Nos centramos en las pseudoclases que solo se pueden aplicar a link con excepción de focus y hover que se puede aplicar a otros elementos.


:Link Este indica el estado cuando a un enlace aun no se le ha hecho click, el que esta por default.
:visited Indica el estado de un enlace cuando se le a hecho click.
:active Este sucede cuando en el momento exacto que le hemos hecho click a un link pues en esa fracción de segundo se puden declarar estilos. Este puede ser util para simular que estamos dando click a un botón, ya que estos estilos se aplican un vez hayamos hecho click en dicho link.
:Hover Son efectos que se aplican una vez pasemos el ratón sobre dicho elemento.
:focus Esto es para cuando el elemento tiene el foco, es decir, esta seleccionado como cuando oprimimos un input del tipo text para ingresar datos, este por ende lo seleccionamos y tiene dicho foco el cual podemos modificar con css.

Botón
.btn:visited { background-color: currentColor; color: cyan; } .btn:active { color: hsl(180,80%,40%); transform:scale(.95); }

Pseudoclases de estado/ UI States

Todas las clases en este apartado son exclusivamente para formulario con excepción de fullscreen ya que esta es mas técnica, así que no la vamos a ver.

:Enabled Sirve para seleccionar los elementos de un formulario que estan habilitados, que basicamente son todos los elementos por default a no ser que en la etiqueta le agreguemos disabled.
:disabled Selecciona a todos los elementos de un form que tengan el atributo disabled.
:checked Hace referencia a los checkbox el cual indica que si el checkbox es seleccionado aplica los siguientes estilos. Con este podemos hacer muchísimos efectos.
:in-range Estos sirven para los input de tipo numero el cual al indicarle un rango de digitos este al estar en rango queremos aplicarle estilos, bueno esto se consigue con in-range.
:out-of-range Seleccionamos el input de tipo numero cuando se sale del rango establecido en etiqueta.
:required Seleccionamos a los inputs que se les especifico en su etiqueta el atributo required, este sirve para aplicar un after o before que mencione que tiene que llevarse el campo o simplemente no es valido.
:optional Indica que los elementos input que sean opcionales se seleccionen con esta pseudoclase. El opcional es el atributo que esta por defecto.
:valid Indica que campos son validos.
:invalid indica que los campos no son validos.
:read-only Indica los campos que les especificamos que solo son de lectura.
:read-write Indica los campos que son de lectura y escuritura. En las etiquetas este seria el por default.

Ejemplo


.input--enabled:enabled { background: red; border:1px solid #000; } .input--disabled:disabled { background: hsla(60,100%,50%,.5); border:1px solid #000; } .input--checkbox:checked + .label{ background-color: red; font-size: 2em; } .input--in-range:in-range { background-color: hsla(35,100%,60%,1); } .input--out-range:out-of-range { background-color: hsl(0,100%,67%); } .input--optional:optional { background-color: hsl(130,100%,70%); } .input--required:required { background-color: hsl(190,100%,70%); } .input--valid:valid { background-color: hsl(130,100%,70%); } .input--invalid:invalid { border-color: hsl(0,100%,55%); } .input--invalid:focus { box-shadow: none; outline: 0 none; }

Pseudo clases del tipo estructura

:root Representa la raiz del documento (html) usando una pseudoclase tenemos mas especificidad que si usamos html, es decir, :root y html simbolizan lo mismo pero :root tienen mas valor en el código. El mejor caso del uso de este es para el empleo de variables.
:empty Esta selecciona las etiquetas que no tienen contenido en el html. Esto puede ser útil para cuando trabajemos con un CMS o con js y que es obligatorio tener una etiqueta para que interactue en el código para después usarla, pero por cascada o un detalle así vemos que dicha etiqueta se ve y no queremos eso podemos quitarle estilos con esta pseudoclase.
Nota: Con la anterior pseudoclase igualmente podemos emplear la propiedad ALL la cual hace que todo lo que se le haya aplicado antes se reseten con el valor de initial, inherit y unset, las unicas propiedad que son exepciones a esta son direction y unicode-bidi ya que estas son fundamentales en idiomas que se leen de derecha a izquierda.

-child (se aplican indiferentemente el tipo de etiqueta)
:first-child Selecciona el primer hijo de un papá, si solo tiene un hijo aun así seleccionara este.
:last-child Selecciona el ultimo hijo del papá, si solo tiene un hijo aun así seleccionara este.
:nth-child() Selecciona al N hijo que este entre primero y el ultimo de sus hermanos.
A este ultimo igualmente podemos poner diferentes valores entre el paréntesis como:
odd que indica que seleccionara los hijos impares.
even indica que seleccione los hijos pares.
Números enteros para indicar la posición del hijo como pude ser 1,2,3...
Ecuaciones En este caso utilizamos una variable llamada n el cual es un multiplicador. El valor de N empieza en 0 y termina cuando ya no tenga elementos que seleccionar o N (0-N). La lógica va así: Si ponemos nth-child(3n) le estamos indicando que queremos que seleccione los hijos que sean resultado de la multiplicación del bucle de 3n; entendiendo mejor, le estamos indicando que seleccione al hijo que resulte de la multiplicación de 3 * 0, 0 ya que el primer valor de N es 0, se repite el bucle pero ahora n vale 1 así que ahora multiplicaremos 3 * 1 que dará 3 así que se seleccionara el hijo en la posición 3, posteriormente n vale 2 así que seria 3 * 2 que da 6 así que seleccionara el hijo num 6, por ultimo n vale 3 por lo que seria 3 * 3 que da 9 y así sucesivamente hasta que se terminen los hijos.
Otro caso en las ecuaciones es usar n + 3 por dar un ejemplo que dara lo mismo, básicamente estamos diciendo que a n se le sume en lugar de multiplicar siguiendo la misma lógica que ya describimos.
nth-last-child Selecciona la posición de los hijos indicada pero contando desde abajo hasta arriba.
only-child Solo selecciona si solo hay un hijo, de lo contrario no se aplicara.

!NOTA: La pseudoclase child se va aplicando a los hijos, si le especificamos un selector este seleccionara a los hijos que sean de dicho selector. Explicando mejor, si dentro del div(papá) tenemos otros 2 divs (hijo 1 y hijo 2) y otros parrafos, al momento de usar first-child estariamos seleccionado a el primer hijo, en este caso el primer div, es decir, no le importa el selector en si, si no el hijo, el selector nos indica que si dicho hijo no es igual al selector entonces este se ignorara.

-type (a este si le importa que tipo de etiqueta, clase, id [selector] se emplee, a diferencia de child que solo le basta con que sea un hijo.)
:First-of-type Selecciona al primer hijo del tipo especificado.
:last-of-type Selecciona al ultimo hijo del tipo especificado.
nth-of-type Selecciona n posición de hijo del tipo especificado.
nth-last-of-type Selecciona n posición de hijo de abajo hacia arriba del tipo especificado.
:only-of-type Solo selecciona al único hijo del tipo especificado.

!NOTA: Este puede llegar a ser mucho mas util si dentro del papá tenemos varios hijos pero solo queremos seleccionar a los hijos que tengan una clase o una etiqueta especifica; a diferencia de child que empieza a contar a partir del primer elemento (son) que esta dentro den contenedor (dad).
