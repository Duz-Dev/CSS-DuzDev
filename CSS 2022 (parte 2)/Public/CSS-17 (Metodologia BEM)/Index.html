Metodología BEM

BEM

¿Qué es una metodología?

Una metodología en css es una serie de consejos para estructurar muestras clases de una forma sencilla, escalable y reutilizable. Con escalable nos referimos a que si ya tenemos código css hecho siguiendo esta metodología podremos incorporar nuevos elementos sin que alteren el layout o que destruyan nuestra estructura.

y ¿qué es BEM?

BEM es una de las metodologías mas utilizadas en el mundo. Sus siglas provienen de Block, element, modifier, esto significa que estructuraremos de tres formas distintas en función de el comportamiento que tome los elementos.

  • Bloques:

    Los bloques se suelen denominar con una sola palabra o con dos palabras separadas con un guion si puede existir conflicto de estilos, estos son los elementos que serán papás de una estructura con un orden por ejemplo un nav seria ya un elemento bloque porque sabemos que dentro de este abra elementos. Estos como ya mencionamos se escriben con un solo nombre main o con dos si sabemos que tenemos mas de un bloque que empieza con este nombre como un main para el header y un main para el footer, en este caso en el footer se llamaria main-footer por dar un ejemplo.

  • Elementos:

    Los elementos se nombran con el nombre del bloque al que pertenecen y su nombre con dos guiones bajos __, ejemplo main-gallery__img.

  • Modificadores:

    Los modificadores se nombran con el nombre del bloque al que pertenecen y su modificador con dos guiones medios --. Estos para dar un ejemplo son bloques a los que les queremos cambiar propiedad o asignarle otra, para ello usamos el nombre de dicho bloque más los guiones medios y el nombre de dicha modificación, ejemplo .menu__item--active

!Ejemplos de nombre de bloque .menu / .main-menu .gallery / .main-gallery
!Ejemplos de Elementos .menu__item / .main-menu__link .gallery__img / .main-gallery__item
!Ejemplos de modificadores .menu__item--active .gallery__item--special

Algo con el que hay que saber si es la primera vez que usa BEM es que no se tienen que crear nietos ya que esto es redundante, si ya sabes que es parte de dicho bloque ¿porque especificar que esta dentro de otro elemento que ya sabemos que esta dentro del bloque?, en fin, para ello solo usamos el nombre del bloque más en nombre de elemento y ya esta, da igual si este elemento esta dentro de otro elemento la regla sigue siendo la misma.
